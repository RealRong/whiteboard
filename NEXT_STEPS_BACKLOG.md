# 可做事项清单（暂不实现）

> 目的：把下一步可做的工作列成清单，便于排期与优先级评估。

## 一、插件/注册表体系强化
- 插件依赖与版本约束检查（requires + semver）
- 插件热加载与卸载恢复（运行时切换能力）
- 插件能力声明与权限控制（capabilities）
- 插件配置存储与加载（插件级配置面板）
- 插件冲突检测（同名注册、命令覆盖）

## 二、Schema 与校验能力扩展
- 字段类型扩展：array/object/date/slider/file/selector
- 复杂校验规则：正则、范围、跨字段约束
- 字段联动：条件显示/隐藏、动态 options、依赖计算
- Schema 版本迁移与变更兼容策略
- 批量默认值与模板应用（批量节点/边）

## 三、属性面板与编辑体验
- 右侧栏面板系统（面板栈/多卡片）
- 自定义控件注册（业务控件、远程数据选择器）
- 面板布局配置（分组、折叠、搜索）
- 表单校验提示与错误聚合
- 编辑历史回滚与应用按钮（apply/cancel）

## 四、节点/边类型体系
- Edge 类型注册表（可自定义路由与渲染）
- Node 模板库：工业/流程/图标/设备
- 端口系统：端口类型、方向、容量限制
- 节点行为扩展（拖拽规则、吸附规则）

## 五、数据绑定与实时态势
- 节点/边绑定点位（tag）
- 实时刷新策略（推送/轮询）
- 状态驱动样式规则（告警、阈值、健康度）
- 统一的数据适配层（可扩展协议）

## 六、性能与大图支持
- 空间索引（Quadtree/R-tree）
- 视口裁剪与虚拟化渲染
- 局部增量重绘与缓存
- 交互降级策略（拖拽时简化路径）
- 性能指标面板（FPS/耗时采样）

## 七、导入导出与集成
- JSON Schema 标准化与版本化
- 增量保存与差异导出
- SDK 嵌入能力（iframe/微前端）
- 事件桥接与外部系统联动

## 八、协作与权限（可选）
- 多用户协作（锁定/OT/CRDT）
- 角色权限与操作审计
- 只读审阅与批注模式

## 建议优先级
1. Schema 扩展与属性面板增强
2. 插件体系稳定性（依赖/冲突/配置）
3. 性能与大图支持
4. 数据绑定与实时态势
5. SDK 与集成能力
